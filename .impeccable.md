# Odin — Design Context

## Design Context

### Users
Open-source project used by developers and general public. Usage context: person wants to send or receive a file quickly — low friction, no account. Sender and receiver may be on different devices (desktop sender, mobile receiver and vice versa). Mobile is a first-class use case.

### Brand Personality
**Bold. Sharp. Trustworthy.**
- Bold: strong visual identity, high-contrast, confident typography
- Sharp: precise, no decoration for decoration's sake, everything earns its place
- Trustworthy: encryption-first, open-source, nothing hidden

Emotional goal: feels powerful and purposeful without being intimidating. Like a well-made tool.

### Aesthetic Direction
Dark-only. Purple (`#7D5DEC`) as the single dominant accent — not diluted across the UI, used for primary actions and brand moments.
Near-black backgrounds (`#151515`, `#111111`). Off-white text (`#fafafa`). Cards at `#1E1E1E`.
Font: Inter — already in use, appropriate for a developer/utility product.
No gradients on text. No glassmorphism. No glow effects.
Bold typography hierarchy: oversized headlines create drama; body text is restrained.
Background illustration (odin_bg) used as a brand texture element, not decoration.

### Design Principles
1. **Full-screen on mobile** — no floating cards, no scaled-down desktop layouts. Each screen owns the full viewport.
2. **Thumb-first CTA placement** — primary actions anchored to the bottom of the screen within easy reach.
3. **One job per screen** — Home sends or receives. Upload shows progress or shares the token. Download accepts a token and downloads. No scope creep.
4. **Typography does the heavy lifting** — big, bold headlines create impact without needing illustrations. Use the odin_bg illustration as texture, not hero.
5. **Purple is precious** — only on primary CTAs and active/success states. Never decorative.

### Platform Strategy
- Mobile (`< 600px logical width`): full-screen native layouts, bottom-anchored CTAs, large touch targets (min 52px height)
- Desktop (`≥ 600px`): existing card-based centered layout, unchanged
- Breakpoint check: `MediaQuery.of(context).size.width < 600`
